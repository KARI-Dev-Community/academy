# Community Guidelines

Welcome to the Kari Dev community. This document outlines how we interact and support each other.

## Our Values

- **Learning first**: Everyone is here to learn, including maintainers
- **Respect**: Treat everyone with kindness regardless of experience
- **Collaboration**: We build better things together
- **Growth**: We celebrate progress, not perfection

## Communication Channels

### GitHub Discussions

Use GitHub Discussions for:
- Questions about specific projects
- Feature requests and ideas
- Show and tell your projects

### Issues

Use Issues for:
- Reporting bugs
- Requesting documentation updates
- Suggesting new projects

### Pull Requests

Use Pull Requests for:
- Code contributions
- Documentation improvements
- Bug fixes

## Getting Help

### Before Asking

1. Search existing issues and discussions
2. Read the relevant documentation
3. Try to solve it on your own for at least 15 minutes
4. Prepare what you have tried

### How to Ask Good Questions

```markdown
## What I am trying to do
Brief description of your goal

## What I tried
- Step 1
- Step 2

## What happened
Error message or unexpected behavior

## What I expected
What should have happened

## My environment
- OS: Ubuntu 22.04
- Node version: 18.17.0
- Project: project-02-todo-app
```

## Giving Help

### As a Community Member

- Answer questions when you can
- Share resources you found helpful
- Celebrate others' progress

### As a Mentor

- Review pull requests within 3 business days
- Provide constructive, actionable feedback
- Focus on teaching, not just fixing
- Be patient with beginners

## Recognition

We recognize community contributions through:

- **Merged PRs**: All contributions are credited in commit history
- **Discord roles**: Active helpers get special recognition
- **Mentor path**: Outstanding contributors can become mentors
- **Featured projects**: Great work is showcased in documentation

## Reporting Issues

If you witness behavior that violates our Code of Conduct:

1. Reach out to a maintainer privately
2. Describe the behavior and context
3. Suggest how you would like it handled

All reports are taken seriously and handled confidentially.

## Community Calendar

- **Weekly**: Mentor office hours (Wednesdays 7pm UTC)
- **Monthly**: Community showcase and feedback session
- **Quarterly**: Roadmap review and planning
