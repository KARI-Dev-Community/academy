# Career Guides

Building a career in tech requires technical skills, soft skills, and strategic networking.

## Technical Interviews

### Common Interview Types

1. **Coding challenges** - Algorithms and data structures
2. **System design** - Design scalable systems
3. **Behavioral** - Past experiences and teamwork
4. **Take-home projects** - Build something over days/weeks

### Algorithm Practice

Focus on these patterns:
- Two pointers
- Sliding window
- Depth-first search (DFS)
- Breadth-first search (BFS)
- Dynamic programming

### System Design Basics

When designing a system, consider:

1. **Requirements**: What does the system need to do?
2. **Scale**: How many users? How much data?
3. **Components**: APIs, databases, caching, queues
4. **Data flow**: How does data move through the system?
5. **Tradeoffs**: What are we optimizing for?

## Building a Portfolio

A portfolio shows your skills through real projects.

### What to Include

1. **3-5 strong projects** - Quality over quantity
2. **Live demos** - Deploy your projects
3. **GitHub repos** - Clean code with READMEs
4. **Blog posts** - Write about what you learn
5. **Contributions** - Open source work counts

### Project README Template

```markdown
# Project Name

Brief description of what it does.

## Tech Stack
- React
- Node.js
- MongoDB

## Features
- Feature 1
- Feature 2

## How to Run
npm install
npm run dev
```

## Resume Writing

### Tips for Developers

- Lead with impact, not duties ("Built feature that increased engagement by 30%")
- Include metrics and numbers
- List technologies used
- Keep it to one page for early career
- Link to GitHub and portfolio

## Networking

### Where to Connect

- GitHub: Contribute and engage with repos
- LinkedIn: Share projects and learning
- Twitter/X: Post about your learning journey
- Discord/Slack: Join developer communities
- Meetups: Attend local or virtual events

### How to Network Effectively

1. Ask specific questions, not just "can you help me?"
2. Share what you are building
3. Offer to help others when you can
4. Follow up on conversations
5. Be consistent, not sporadic

## Continuous Learning

### Learning Strategies

- **Build projects**: The best way to learn is by doing
- **Teach others**: Writing and explaining deepens understanding
- **Read code**: Study well-written open source projects
- **Stay curious**: Explore new technologies regularly
- **Set goals**: Have a 30/60/90 day learning plan

### Recommended Cadence

- **Daily**: Code for at least 30 minutes
- **Weekly**: Complete one project milestone or tutorial
- **Monthly**: Build and ship a complete project
